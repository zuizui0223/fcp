# Blinded review protocol for the 34 baseline classifications

## Purpose

The current within- and among-population states are source-traceable, rule-derived screening labels. This protocol provides a practical route to independent human verification without exposing the current rule label during the first-pass decision.

## Files

- `docs/supporting/jbi_table_s18_blinded_classification_review.csv` — reviewer-facing sheet; current rule labels are hidden.
- `docs/supporting/jbi_table_s19_rule_classification_key.csv` — rule-derived labels and signal diagnostics; keep closed until the blinded review is complete.
- `docs/supporting/jbi_table_s6_frozen_classification_manifest.csv` — current frozen analysis manifest.
- `evaluate_jbi_completed_classification_review.py` — validates completed review copies and computes agreement, Cohen's kappa and rule-label discrepancies without changing analysis inputs.

## Operational labels

Use exactly one label for each species:

- `within_population`: the retained source explicitly supports coexistence of at least two discrete natural flower-colour variants within at least one population.
- `among_population`: the retained source supports geographic or among-population differentiation, but does not support local coexistence in the reviewed evidence.
- `mixed`: the reviewed evidence supports both local coexistence and geographic differentiation.
- `unclear`: the reviewed evidence is insufficient to distinguish the preceding states.

Do not infer local coexistence from the word `polymorphism` alone. Do not infer geographic differentiation merely because a study sampled more than one locality. Classify the spatial arrangement explicitly supported by the source.

## Review procedure

1. Make a copy of S18 for each independent reviewer.
2. Keep S19 unavailable until all first-pass labels are frozen.
3. For each row, inspect the classification source identifier. The queue excerpt is a navigation aid, not a substitute for the cited source when it is incomplete or when `source_match_status` reports a mismatch.
4. Fill `reviewer_label`, `reviewer_name_or_initials`, `review_date` and a concise `review_notes` justification.
5. Do not inspect climatic niche values, model results or the rule label while classifying.
6. Freeze each completed reviewer sheet before opening S19.
7. Compare reviewers with each other and with the rule key.
8. Resolve disagreements by a documented consensus meeting or a named adjudicator who inspects the source without reference to climatic results.
9. Record every accepted change in the classification correction log; never silently overwrite the frozen manifest.
10. Rerun the complete analysis and submission validators after any classification change.

## Evaluate completed sheets

For one completed reviewer sheet:

```bash
python evaluate_jbi_completed_classification_review.py \
  --review path/to/reviewer_1_completed.csv
```

For two independent reviewers:

```bash
python evaluate_jbi_completed_classification_review.py \
  --review path/to/reviewer_1_completed.csv \
  --review path/to/reviewer_2_completed.csv
```

Outputs are written by default to `analysis_outputs/jbi_classification_review/`:

- `jbi_classification_review_comparison.csv`
- `jbi_classification_review_summary.json`

The evaluator requires all 34 labels, reviewer identity or initials and review dates. For two or more reviewers it reports raw agreement and Cohen's kappa. It surfaces reviewer disagreements and consensus disagreements with the frozen rule label. It never changes S6, S8, model data or manuscript results automatically.

## Minimum acceptable review designs

Preferred:

- two independent reviewers for all 34 species;
- raw agreement and Cohen's kappa reported;
- disagreements resolved by consensus or a third reviewer.

Acceptable with explicit limitation:

- one reviewer verifies all 34 species against the cited sources;
- no independent duplicate review or agreement statistic;
- manuscript states this fact directly.

Current repository-supported state before completion of S18:

- labels were generated by deterministic rules;
- mixed and unclear rows were routed to a manual-review queue;
- the resolved queue retains `review_status = unreviewed`;
- completed independent human review is not documented.

## Decision outcomes

- Human label agrees with rule label: retain the classification and record verification.
- Human label is `mixed` or `unclear`: remove the species from the binary baseline analysis unless additional source evidence resolves it.
- Human within/among label disagrees with the rule label: inspect the full source, document the reason and adjudicate before changing the manifest.
- Classification source differs from the queue's best source: review the classification source first; treat the queue excerpt only as supplementary context.

## Submission gate

Before claiming human-verified classifications, the repository must contain:

- completed reviewer sheet(s);
- reviewer identities or initials and dates;
- the agreement or single-reviewer statement;
- a disagreement-resolution record;
- an updated correction log;
- rerun model outputs and a passing strict release.

Until then, the manuscript must retain the phrase `source-traceable, rule-derived classifications` and must not describe the baseline labels as independently human-screened or manually validated.
